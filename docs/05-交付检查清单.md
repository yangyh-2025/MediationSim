- [x] `.env` — API 密钥配置（3 个变量：OPENAI_API_KEY / OPENAI_BASE_URL / LLM_MODEL）

- [x] `.gitignore` — 排除 `.venv/`、`__pycache__/`、`*.db`、`.env`、`node_modules/`

### 配置与数据模型
- [x] `config.py` — 全局配置类，7 个实验条件，`.env` 加载，cost-saving 默认值
- [x] `models/schemas.py` — 15 个 Pydantic v2 模型，含 NaN 安全序列化、条件校验

### 数据库层
- [x] `db/database.py` — SQLite 异步管理，5 张表，自动创建和索引
- [x] `db/queries.py` — 16 个查询函数，完整 CRUD + 聚合，NaN 安全写入

### 智能体模块（5 类 + 基类）
- [x] `agents/base.py` — 抽象基类，缓存优化（system prompt 不可变）
- [x] `agents/strong_party.py` — 强方（参照以色列），权力型话语
- [x] `agents/weak_party.py` — 弱方（参照埃及），权利型话语
- [x] `agents/mediator.py` — 调停者，3 种偏见模式，3 层附带支付决策
- [x] `agents/domestic_audience.py` — 国内观众，政治可接受性评估
- [x] `agents/evaluator.py` — 独立评估者，6 维量化评估

### LLM 客户端 + 日志系统
- [x] `llm/client.py` — OpenAI 兼容接口，结构化输出，自动重试，缓存预热，**全量日志钩子**
- [x] `llm/logger.py` — 全量 LLM 调用 JSONL 日志系统，线程安全，按实验 + 日期分文件

- [x] `engine/negotiation.py` — 8 步谈判状态机，Gini 计算，call_context 注入，experiment_id 传播
- [x] `engine/scheduler.py` — 实验调度，并行条件执行，3 层迭代，缓存预热，缓存统计打印
- [x] `engine/orchestrator.py` — 评估协调，快/中/全局三层

### 统计分析（6 种检验，H1-H4 全覆盖）

- [x] `analysis/mediation.py` — Bootstrap 中介（5000 次），偏差校正百分位 CI，Sobel 检验
