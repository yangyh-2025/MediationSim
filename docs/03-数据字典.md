## 数据库表结构

### experiments 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PK | UUID 实验ID |
| name | TEXT | 实验名称 |
| status | TEXT | draft / running / paused / completed / failed |
| config_json | TEXT | ExperimentConfigIn 的 JSON 序列化 |
| total_runs | INTEGER | 总运行次数（条件数 × 每条件运行数） |
| completed_runs | INTEGER | 已完成运行次数 |
| created_at | TEXT | ISO 8601 时间戳 |
| updated_at | TEXT | ISO 8601 时间戳 |

### runs 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PK | UUID 运行ID |
| experiment_id | TEXT FK | 关联实验 |
| condition_code | TEXT | 条件码（见上表） |
| run_index | INTEGER | 该条件下的运行序号 (0-29) |
| status | TEXT | pending / running / completed / failed |
| rounds_completed | INTEGER | 完成谈判轮数 |
| agreement_reached | INTEGER | 0=未达成, 1=达成 |
| agreement_gini | REAL | 协议基尼系数 (0-1) |
| side_payment_used | REAL | 附带支付总使用额度 |
| result_json | TEXT | RunResult 完整 JSON |
| created_at | TEXT | ISO 8601 时间戳 |

### rounds 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PK | UUID |
| run_id | TEXT FK | 关联运行 |
| round_number | INTEGER | 轮次编号 (1-10) |
| proposal_json | TEXT | 调解人提案 JSON |
| strong_response_json | TEXT | 强方回应 JSON |
| weak_response_json | TEXT | 弱方回应 JSON |
| domestic_scores_json | TEXT | 国内观众评分 JSON |
| agreement_reached | INTEGER | 该轮是否达成协议 |
| created_at | TEXT | ISO 8601 时间戳 |

### evaluations 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PK | UUID |
| experiment_id | TEXT FK | 关联实验 |
| batch_start | INTEGER | 评估批次起始运行序号 |
| batch_end | INTEGER | 评估批次结束运行序号 |
| condition_code | TEXT | 评估对应的条件 |
| dimensions_json | TEXT | 六维评估分数 JSON |
| overall_score | REAL | 综合评分 (0-10) |
| adjustments_json | TEXT | 参数调整建议 JSON |
| created_at | TEXT | ISO 8601 时间戳 |

### analysis_results 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PK | UUID |
| experiment_id | TEXT FK | 关联实验 |
| hypothesis | TEXT | 假设编号 (H1/H2/H3/H4) |
| test_name | TEXT | 检验方法名称 |
| test_statistic | REAL | 检验统计量 |
| p_value | REAL | p值 |
| effect_size | REAL | 效应量 |
| confidence_interval_json | TEXT | 95%置信区间 JSON |
| significant | INTEGER | 0=不显著, 1=显著 |
| result_json | TEXT | 完整结果 JSON |
| created_at | TEXT | ISO 8601 时间戳 |

## 六维评估指标

| 维度编号 | 维度名称 | 评分范围 | 评估内容 |
|----------|----------|----------|----------|
| 1 | 外部效度 | 0-10 | 模拟行为与真实国际谈判的相似度 |
| 2 | 内部一致性 | 0-10 | 智能体行为是否符合预设角色 |
| 3 | 行为合理性 | 0-10 | 谈判策略是否符合基本逻辑 |
| 4 | 策略多样性 | 0-10 | 同条件下模拟结果的变异性 |
| 5 | 随机充分性 | 0-10 | 结果差异是否由随机性驱动 |
| 6 | 操作检查 | 0-10 | 实验操控是否成功 |

## 文件存储结构

```
data/
├── mediation_sim.db          # SQLite 数据库（所有结构化数据）
├── experiments/               # 实验级 JSON 备份
│   └── {experiment_id}/
│       ├── config.json        # 实验配置
│       └── runs/              # 运行结果备份
│           └── {run_id}.json
├── evaluations/               # 评估报告 JSON
│   └── {evaluation_id}.json
├── results/                   # 分析输出
│   ├── survival_curves/       # 生存曲线图 PNG
│   └── analysis_{exp_id}.json # 分析结果汇总
└── logs/                      # LLM 调用全量日志
    └── {experiment_id}/
        └── llm_calls_YYYY-MM-DD.jsonl   # 按日分文件
```

### LLM 调用日志字段 (JSONL)

每行一条完整的 API 调用记录：

| 字段 | 类型 | 说明 |
|------|------|------|
| `timestamp` | string | UTC ISO 8601 时间戳 |
| `experiment_id` | string | 所属实验 ID |
| `agent_name` | string | 智能体名称（StrongParty/WeakParty/Mediator/DomesticAudience-强/弱/Evaluator） |
| `condition_code` | string | 实验条件码 |
| `round_number` | int | 谈判轮次 |
| `run_id` | string | 模拟运行 ID |
| `model` | string | 使用的模型名 |
| `temperature` | float | 温度参数 |
| `duration_ms` | float | 调用耗时（毫秒） |
| `cache_hit_tokens` | int | 缓存命中 token 数 |
| `cache_miss_tokens` | int | 缓存未命中 token 数 |
| `messages` | list[dict] | **完整提示词**（system + user，原文全量保存） |
| `response_text` | string | **LLM 原始响应全文** |
| `parsed_output` | object/null | 解析后的结构化输出 |
| `error` | string/null | 错误信息（如有） |
