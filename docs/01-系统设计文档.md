# 偏见调停多智能体模拟系统 — 设计文档

## 1. 项目概述

基于 LLM 多智能体的国际冲突调停模拟系统，以戴维营协议为经验锚点，通过 `2×3 全因子组间实验 + 1 组参照条件`（共 7 条件，210 次模拟），检验偏见调停有效性假说及其附带支付中介机制。

**技术栈：** FastAPI + React + SQLite + JSON 文件 + OpenAI 兼容接口  
**语言：** Python 3.10+（后端），TypeScript（前端）  
**代码量预估：** 后端 ~2000 行 Python，前端 ~1500 行 TS/JSX

## 3. 后端架构

### 3.1 核心数据流

```
用户配置 → ExperimentScheduler → [7条件 × 30次] 并行
                │
                ▼
         NegotiationEngine (每次模拟)
                │
    ┌───────────┼───────────┐
    ▼           ▼           ▼
  Agent.S   Agent.W   Agent.M  ←→ Agent.D
    │           │           │
    └───────────┼───────────┘
                ▼
        协议达成/破裂判定
                │
                ▼
        Evaluator (批次完成后)
                │
                ▼
        参数迭代建议 → Scheduler
                │
                ▼
        所有数据 → SQLite + JSON文件
                │
                ▼
        Statistics 模块 → 前端展示
```

### 3.2 智能体模块 (`agents/`)

所有智能体继承 `BaseAgent`：

```python
class BaseAgent:
    name: str
    role: str
    system_prompt: str
    output_schema: Type[BaseModel]   # Pydantic schema 约束输出
    llm_client: LLMClient

    async def act(self, context: NegotiationContext) -> BaseModel:
        """接收谈判上下文，返回结构化动作"""
```

**五类智能体：**

| 智能体 | 类名 | 核心产出 |
|--------|------|----------|
| 强方 S | `StrongParty` | 立场声明、反提案、接受/拒绝决定 |
| 弱方 W | `WeakParty` | 立场声明、反提案、接受/拒绝决定 |
| 调停者 M | `Mediator` | 协议草案（含附带支付方案） |
| 国内观众 D | `DomesticAudience` | 政治可接受性评分 [0, 1] |
| 评估者 E | `Evaluator` | 6 维评估报告 JSON |

### 3.3 谈判引擎 (`engine/`)

**8 步流程（状态机）：**

```
Step1 初始化 → Step2 立场声明 → Step3 调停提案 →
Step4 国内观众评估 → Step5 回应与反提案 →
Step6 协议判定 → Step7 迭代控制 → Step8 评估记录
                                              │
                                              ▼ (若未达成且轮次 < 10)
                                        返回 Step3
```

**协议达成条件（二选一）：**
- 双方均接受当前草案
- 任何一方的不接受理由被调停者的附带支付方案化解

**结构化输出验证：**
- 领土比例合计不超过 100%
- 附带支付额度不超过调停者资源上限
- 所有数值字段在合法范围内

### 3.4 实验调度 (`engine/scheduler.py`)

**7 个实验条件：**

| 条件代号 | 实力不对称度 (AR) | 调停者类型 (b值) | 运行次数 |
|----------|-------------------|------------------|----------|
| H-PS | 高 (3:1) | 亲强 (+0.7) | 30 |
| H-N | 高 (3:1) | 中立 (0.0) | 30 |
| H-PW | 高 (3:1) | 亲弱 (-0.7) | 30 |
| L-PS | 低 (1.5:1) | 亲强 (+0.7) | 30 |
| L-N | 低 (1.5:1) | 中立 (0.0) | 30 |
| L-PW | 低 (1.5:1) | 亲弱 (-0.7) | 30 |
| CD | 戴维营 (2:1) | 亲强 (+0.7) | 30 |

**调度策略：**
- 同条件实验串行（共享上下文无益处，反而污染）
- 不同条件并行（独立 API session，互不干扰）
- 单次模拟最多 10 轮谈判
- 每 10 次触发快速评估迭代
- 每 30 次触发中期评估迭代

### 3.5 评估系统 (`engine/orchestrator.py` + `agents/evaluator.py`)

**六维评估指标体系：**

| 维度 | 关键指标 | 检测方法 |
|------|----------|----------|
| 外部效度 | 权力/权利话语占比、破裂率与历史分布对比 | 文本模式匹配 + 统计检验 |
| 内部一致性 | 让步率组间差异、提案方向与 b 值一致性 | t 检验 + 相关系数 |
| 行为合理性 | 让步幅度递减趋势、附带支付-僵局关联 | 时间序列检验 |
| 策略多样性 | 基尼系数 SD、轮次 CV、策略类型熵 | 描述统计 |
| 随机充分性 | 同条件协议分布 vs 二项分布 | 拟合优度检验 |
| 操作检查 | 偏见组提案偏向性、不对称度资源差异 | 独立 t 检验 |

**三层迭代：**
1. **快速迭代（每 10 次）**：微调提示词参数（让步阈值 ±0.05、支付倾向 ±0.1）
2. **中期迭代（每 30 次）**：检查策略多样性，必要时补充边界测试
3. **全局迭代（条件切换前）**：审查整体一致性，确保条件间可比

### 3.6 LLM 客户端 (`llm/client.py`)

```python
class LLMClient:
    async def chat(
        self,
        messages: list[dict],
        response_schema: Type[BaseModel] | None = None,
        temperature: float = 0.7,
        max_tokens: int = 3840,
    ) -> BaseModel | str:
        """OpenAI 兼容接口，支持结构化输出"""
```

- 自动重试（最多 3 次，指数退避）
- 结构化输出校验失败时重新请求
- 速率限制检测与等待

### 3.7 统计分析 (`analysis/`)

| 假设 | 主检验方法 | 补充分析 | 效应量 |
|------|-----------|---------|--------|
| H1 偏见主效应 | 独立 t 检验 | Logistic 回归 | Cohen's d |
| H2 协议质量/持久性 | ANOVA + Kaplan-Meier | Cox PH 模型 | η², HR |
| H3 中介效应 | Bootstrap (5000 次) | 禁用支付对照 | 间接效应 CI |
| H4 调节效应 | 双因素 ANOVA | 简单效应分析 | η²_p |

### 3.8 API 路由设计

```

POST   /api/experiments/{id}/pause  暂停实验
POST   /api/experiments/{id}/resume 恢复实验
DELETE /api/experiments/{id}        删除实验

GET    /api/experiments/{id}/runs           获取所有运行记录
GET    /api/experiments/{id}/runs/{run_id}  获取单次运行详情
GET    /api/experiments/{id}/runs/{run_id}/transcript  谈判全文

GET    /api/experiments/{id}/evaluations    获取评估历史
GET    /api/experiments/{id}/statistics     获取统计结果

GET    /api/health                   健康检查
GET    /api/config                   获取当前配置
PUT    /api/config                   更新配置
```

## 6. 关键技术决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| LLM 结构化输出 | Pydantic + `response_format` | 研究明确要求，防止幻觉 |
| 异步并发 | asyncio + `asyncio.gather` | 210 次实验需并行加速 |
| 谈判轮次上限 | 10 轮 | 与提纲一致 |
| LLM 温度 | 默认 0.7，可配置 | 平衡一致性与多样性 |
| 每轮最大 tokens | 3840 | 与研究设计一致 |
| Bootstrap 重抽样 | 5000 次 | 标准中介检验要求 |
| 统计显著水平 | α = 0.05（双尾） | 社会科学标准 |
