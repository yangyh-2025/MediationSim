# 实验操作手册 — 偏见调停多智能体模拟系统

## 如何使用本系统完成完整研究

本手册将研究提纲中定义的 4 个实验阶段逐一映射为系统操作步骤。你无需写任何代码——所有操作均通过前端页面点击完成。

---

## 成本控制说明

为降低 API 调用成本，本方案对研究提纲做如下精简（均在统计功效可接受范围内）：

| 参数 | 提纲原值 | 精简值 | 节省 |
|------|---------|--------|------|
| 每条件运行次数 | 30 | **10** | -67% |
| 最大谈判轮次 | 10 | **8** | -20% |
| 每轮最大 tokens | 3840 | **2048** | -47% |
| Phase 1 预实验 | 35次 | **21次** | -40% |
| Phase 2 主实验 | 210次 | **70次** | -67% |
| Phase 3 中介检验 | 90次 | **30次** | -67% |

**总成本约为原方案的 1/3**。样本量从每组 30 降至 10 后，仍可检测中等及以上效应量（Cohen's d ≥ 0.8），足以完成方法学验证和探索性分析。

## Phase 1：预实验与参数校准

**目的：** 验证智能体行为是否符合预期，校准参数。数据不纳入正式统计。

### 操作

点击左侧 **"实验配置"** → **"创建新实验"**：

| 字段 | 填什么 |
|------|--------|
| 实验名称 | `预实验-参数校准` |
| 温度 | `0.7`（默认） |
| 最大 Tokens | `2048`（默认） |
| 最大谈判轮次 | `8`（默认） |
| **每条件运行次数** | **`3`** ⚠️ 预实验用3次 |
| 附带支付 | **打开** |
| 实验条件 | **全选 7 个** |

条件预览：

| 条件码 | AR | 偏见 b | 运行次数 |
|--------|-----|--------|----------|
| H-PS | 3.0 | +0.7 亲强 | 3 |
| H-N | 3.0 | 0.0 中立 | 3 |
| H-PW | 3.0 | -0.7 亲弱 | 3 |
| L-PS | 1.5 | +0.7 亲强 | 3 |
| L-N | 1.5 | 0.0 中立 | 3 |
| L-PW | 1.5 | -0.7 亲弱 | 3 |
| CD | 2.0 | +0.7 亲强 | 3 |

点击 **"创建实验"** → **"启动"**。

切换到 **"实验监控"** 页，下拉选 `预实验-参数校准`，观察 21 次运行。预实验约需 **5-10 分钟**。

完成后检查评估报告，确认：
- [ ] CD（戴维营参照组）协议率在合理范围
- [ ] 亲强组提案确实偏向强方（操作检查通过）
- [ ] 评估综合分 ≥ 6.0

## Phase 2：主实验 — 偏见效应检验

**目的：** 检验 H1（偏见主效应）和 H4（调节效应）。

### 操作

点击 **"实验配置"** → **"创建新实验"**：

| 字段 | 值 |
|------|-----|
| 实验名称 | `主实验-偏见效应检验` |
| 温度 | `0.7` |
| 最大 Tokens | `2048` |
| 最大谈判轮次 | `8` |
| **每条件运行次数** | **`10`** |
| 附带支付 | **打开** |
| 实验条件 | **全选 7 个** |

预览确认共 70 次。**"创建实验"** → **"启动"**。

在 **"实验监控"** 页跟踪。70 次预计 **20-40 分钟**。

完成后去 **"结果总览"** → 下拉选 `主实验-偏见效应检验` → 点 **"运行统计分析"**。

H1 卡片：看 t 检验的 p 值和 Cohen's d。  
H4 卡片：看交互项是否显著。

## Phase 3：附带支付中介效应检验

**目的：** 禁用附带支付，与 Phase 2 数据合并做 Bootstrap 中介检验（H3）。

### 操作

**关键区别：关闭支付，只选亲强条件。**

| 字段 | 值 |
|------|-----|
| 实验名称 | `对照实验-禁用支付` |
| 温度 | `0.7` |
| 最大 Tokens | `2048` |
| 最大谈判轮次 | `8` |
| 每条件运行次数 | `10` |
| **附带支付** | ⚠️ **关闭** |
| **实验条件** | ⚠️ **仅选 H-PS, L-PS, CD** |

预览确认共 30 次。**"创建实验"** → **"启动"**。

约 **10-20 分钟**完成。

去 **"详细分析"** → **"中介效应"** 标签页，查看 Bootstrap 结果。

**预期：** 禁用支付后协议率显著下降。若 95% CI 不包含 0 → H3 中介效应成立。

---

## Phase 4：协议持久性生存分析

**目的：** 对 Phase 2 中达成协议的案例追加执行期，检验 H2。

### 操作

1. **"详细分析"** → **"生存分析"** 标签页
2. 数据源选 `主实验-偏见效应检验`
3. 点 **"运行持久性分析"**
4. 系统自动筛选达成协议案例，追加 5 轮无调停者的执行期
5. 查看 Kaplan-Meier 生存曲线（3 条线对应 3 种调停者类型）

回到 **"结果总览"**，H2 卡片显示 gini ANOVA + 生存分析 + Cox PH 风险比。

## 四个 Phase 一览

```
Phase 1       Phase 2       Phase 3       Phase 4
21次 预实验 → 70次 主实验 → 30次 禁用支付 → 达成案例×5轮
~5-10分钟     ~20-40分钟    ~10-20分钟     ~10-15分钟
校准参数      H1 + H4       H3             H2
```

| 假设 | 去哪里看 | 关键判断标准 |
|------|---------|-------------|
| H1 偏见主效应 | 结果总览 → H1卡片 | p < 0.05（样本有限时可放宽至 p < 0.10） |
| H2 质量与持久性 | 结果总览 → H2卡片 | η² ≥ 0.06, HR > 1.0 |
| H3 中介效应 | 详细分析 → 中介效应 | Bootstrap 95% CI 不包含 0 |
| H4 调节效应 | 结果总览 → H4卡片 | 交互项 p < 0.05 |

> **注意：** 每组 10 次的样本量下，统计检验的显著性可能受限。这在探索性/方法学验证研究中是可接受的——核心目标是验证整套"案例参数校准→提示工程驱动→评估迭代优化→统计假设检验"方法论管道的可行性。如需严格的假设检验显著性，后续可将每条件运行次数增加至 30。
