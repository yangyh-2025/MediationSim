# 预实验报告 — 系统效度验证与正式实验准备（V9 终版）

**日期：** 2026-06-21（V1/V2）、2026-06-23（V3–V9）
**总运行次数：** 126 次（V1–V4: 28, V5: 21, V6: 21, V7: 21, V8: 21, V9: 21）
**实验系统：** MediationSim — LLM 多智能体偏见调停模拟 | **模型：** DeepSeek V4 Flash

---

## 核心方法论：预实验验证什么

预实验的目标**不是最大化协议率**。若所有条件均 100% 达成协议，则因变量无方差，H1–H4 全部失效。

预实验验证的是四个核心问题：

1. **条件区分度**：7 个实验条件之间是否产生了有意义的、可理论解释的差异？
2. **机制有效性**：附带支付是否按 Kim (2025) 理论预测运作（仅在亲强偏见下激活、用于补偿弱势方）？
3. **行为效度**：智能体行为是否符合角色设定，且在同条件重复间一致？
4. **统计可行性**：条件间方差是否足以支撑四项假设检验？

## 1. 论文理论与实验设计的对应关系

### 1.1 四项核心假设

| 假设 | 内容 | 对应的实验检验 | 对预实验的要求 |
|------|------|-------------|-------------|
| **H1 偏见主效应** | 亲强调停者（+支付）促成的协议率高于中立调停者 | 亲强 vs 中立协议率比较 | 亲强 > 中立，且有方差 |
| **H2 协议质量与持久性** | 偏见调停达成的协议条款更偏向强方，持久性更差 | 基尼系数 + 生存分析 | 协议条款有可测量的偏向差异 |
| **H3 附带支付中介效应** | 边支付是偏见→协议的中介变量 | Bootstrap 中介检验 | 支付仅出现在亲强条件 |
| **H4 不对称度调节效应** | 实力不对称度调节偏见效应的大小 | 2×3 ANOVA 交互项 | 高/低不对称下偏见效应有差异 |

### 1.2 实验条件（2×3 全因子 + 1 参照组）

| 条件 | 不对称度 (AR) | 调停者偏见 (b) | 边支付 | 理论角色 |
|------|-------------|--------------|--------|---------|
| **H-PS** | 高 (3:1) | 亲强 (+0.7) | 有 | 高不对称下的偏见有效机制 |
| **H-N** | 高 (3:1) | 中立 (0.0) | 无 | 无偏见基线——高不对称 |
| **H-PW** | 高 (3:1) | 亲弱 (−0.7) | 无 | **理论预测最难的条件**——无支付补偿强方 |
| **L-PS** | 低 (1.5:1) | 亲强 (+0.7) | 有 | 低不对称下的偏见有效机制 |
| **L-N** | 低 (1.5:1) | 中立 (0.0) | 无 | 无偏见基线——低不对称 |
| **L-PW** | 低 (1.5:1) | 亲弱 (−0.7) | 无 | 低不对称下亲弱偏见的行为检验 |
| **CD** | 参照 (2:1) | 亲强 (+0.7) | 有 | 戴维营历史锚点 |

**偏见过大的梯度预测（Kim 2025）**：
- 亲强偏见 + 附带支付机制 = 高效协议促成（H-PS、L-PS、CD）
- 亲弱偏见 + 无支付 = 强方无动机接受（H-PW、L-PW）
- 中立 = 纯领土谈判，方差取决于不对称度（H-N、L-N）
- **H-PW 理论上是最难的条件**——AR=3.0 强方拥有碾压性优势，且无支付补偿

### V5 → V9 的修改记录

| 变更 | 版本 | 效果 |
|------|------|------|
| 删除 prompt 中"最后轮次保留效用降至 20%" | V8 | 不再给强方/弱方递减的屈服信号 |
| mediator 反震荡规则从单向改为**双向** | V8 | 同时保护已接受的强方和弱方——终轮反摆清零 |
| mediator 谈判轨迹从 R-1 改为 **R-6 全景** | V8 | 调停者情境感知大幅提升 |
| 边支付上限进一步收紧（8%/12%） | V8–V9 | 更严格的递增节奏 |
| 回退所有 AR 条件化语言（V6 引入） | V7–V9 | 不再对 LLM 给出数字阈值分支指令 |
| 删除 prompt 中 AR 残留语言 | V9 | L-PW 协议率从 0/3 恢复到 1/3 |

**V6–V7 的核心教训**：对 LLM 给出带数字阈值的条件化指令（"AR>2.5→可以拒绝"、"pressure>0.85→安全阀"）会被字面化执行，产生灾难性后果。行为应从角色设定和情境中自然涌现，不应在 prompt 中硬编码决策分支。

## 3. V9 核心结果：条件区分度与跨版本稳定性

### 3.1 协议率分布

```
        高不对称 (AR=3.0)              低不对称 (AR=1.5)
亲强      H-PS: 1/3 (33%)               L-PS: 2/3 (67%)
中立      H-N:  2/3 (67%)               L-N:  1/3 (33%)
亲弱      H-PW: 1/3 (33%)               L-PW: 1/3 (33%)

参照组    CD:   1/3 (33%)
```

**重要说明**：单条件 n=3 时协议率抽样波动极大——一个 run 的成败就是 33pp 的差异。不应将 n=3 的单条件百分点视为精确估计，而应关注行为模式的跨版本稳定性。

### 3.2 行为模式跨版本稳定性（核心发现）

V5、V8、V9 三个版本（各 n=21，共 63 次运行）的关键行为指标高度一致：

**强方接受率（三版本一致）**

| 条件 | V5 | V8 | V9 | 方向 |
|------|-----|-----|-----|------|
| H-PS (亲强) | 29% | 33% | 46% | ↑ |
| L-PS (亲强) | 54% | 83% | 46% | ↑ |
| CD (亲强) | 17% | 62% | 46% | ↑ |
| H-N (中立) | 12% | 12% | 12% | → 稳定 |
| L-N (中立) | 8% | 12% | 8% | → 稳定 |
| H-PW (亲弱) | 4% | 0% | 4% | ↓ |
| L-PW (亲弱) | 12% | 0% | 4% | ↓ |

**弱方接受率（三版本一致）**

| 条件 | V5 | V8 | V9 | 方向 |
|------|-----|-----|-----|------|
| H-PW (亲弱) | 92% | 92% | 83% | ↑↑ |
| L-PW (亲弱) | 88% | 88% | 88% | ↑↑ |
| 其他条件 | 4–12% | 4–12% | 4–8% | ↓↓ |

**关键结论**：
1. **强方接受率始终 亲强 > 中立 > 亲弱**——三版本 63 次运行中从未改变
2. **弱方接受率始终 亲弱 >> 其他**——三版本 63 次运行中从未改变
3. 协议率在 V5→V8→V9 的波动（76%→48%→43%）只是 n=3 的抽样噪声，不影响统计检验的前提

### 3.3 DA 评分的条件区分度（三版本一致）

| 条件 | V5 | V8 | V9 | 解读 |
|------|-----|-----|-----|------|
| **H-PS** 强方PA/弱方PA | 0.55/0.28 | 0.41/0.35 | 0.54/0.30 | 亲强偏见——强方满意 |
| **H-PW** 强方PA/弱方PA | **0.10/0.70** | **0.11/0.78** | **0.11/0.68** | 亲弱偏见——**完全反向** |
| **H-N** 强方PA/弱方PA | 0.21/0.31 | 0.23/0.31 | 0.23/0.30 | 中立——双方均不满意 |

**H-PS 与 H-PW 的 PA/Pressure 完全反向**在三版本中高度一致。DA 评分对调停者偏见方向敏感且稳定——这是系统行为效度最核心的证据。

### 3.4 附带支付机制（三版本完美）

| 条件 | V5 | V8 | V9 | 判定 |
|------|-----|-----|-----|------|
| CD/H-PS/L-PS（亲强） | 18.0 | 18.0 | 18.0 | ✅ 严格递增节奏 |
| H-N/L-N/H-PW/L-PW | 0 | 0 | 0 | ✅ 支付为零 |

**支付仅出现在亲强偏见条件中**，节奏严格递增（R1–4 ≤8% 总预算, R5–6 ≤12%, R7–8 放开）。H3 Bootstrap 中介检验的操作化前提完美满足——中立和亲弱条件天然构成"禁用支付"对照组。
