## 3. 关键系统缺陷诊断

### 3.1 🔴 致命缺陷：DomesticAudience 输出 Schema 错乱

**现象**：数据库中所有 `domestic_scores_json` 的 `political_acceptability` 和 `pressure_level` 均为 0.00。

**根因**：DeepSeek V4 Flash 在响应 `DomesticAudience` 的提示词时，出现了**输出 Schema 混淆**：

| 输出类型 | 占比 | 说明 |
|----------|------|------|
| `AgentResponse`（错误） | 42.6%（49/115） | 含 action、counter_proposal、reasoning |
| `Proposal`（错误） | 18.3%（21/115） | 含 territory_split、resource_allocation |
| `DomesticScore`（正确） | 39.1%（45/115） | 含 political_acceptability、pressure_level |

当 LLM 输出 `AgentResponse` 或 `Proposal` 的 JSON 对象时，它不包含 `DomesticScore` 的必需字段（`political_acceptability`、`pressure_level`）。Pydantic 校验失败后，系统经过 3 次重试仍然失败，最终返回一个**字段全为默认值的 `DomesticScore` 对象**（均为 0.0）。

**影响链**：
```
DomesticAudience 输出错误 Schema
    → DomesticScore 解析失败/字段为空
    → strong_party.respond_to_proposal() 的 pressure_note 始终为空
    → 强方/弱方在 respond_to_proposal 时看不到真实的国内压力反馈
    → 但强方/弱方在第一阶段 act() 的提示词中仍然包含"国内鹰派议会对领土损失高度敏感"等硬约束
    → 智能体在推理文本中自述"国内压力 0.95-1.00"，但这来自提示词文本暗示而非 DomesticAudience 的计算结果
    → 谈判反馈环断裂：双方无法根据真实国内压力动态调整立场
```

### 3.2 🟡 严重问题：智能体拒绝阈值过高

分析 56 轮谈判的行为模式：

1. **任何一方的 `accept` 率极低**：强方 7.1%，弱方 16.1%。绝大多数回应为 `counter_proposal`（75-86%），实质上是软拒绝。

2. **双方提示词中的"硬约束"导致不可妥协**：
   - 强方提示词："国内政治压力超过 0.7 时，几乎无法做出任何让步"
   - 弱方提示词："国内政治压力超过 0.7 时，领导人几乎无法接受任何不收复领土的协议"
   - 双方同时具有强国内政治敏感性 → 双向僵局

3. **调停者边支付几乎无效**：
   - 仅亲强条件（PS）有边支付启用（3/7 条件）
   - 边支付金额要么为 0，要么极小（0.03）
   - 弱方反复在其逻辑中表明"边支付不能替代领土主权"

4. **缺乏"最后轮次效应"**：即使到第 8 轮（最后一轮），双方仍不妥协。提示词规定了明确的"无协议成本"（强方保留效用=初始的 60%），但智能体的决策文本中几乎不考虑这个成本。

## 5. 设计优化方案

### 5.1 🔴 优先级 P0：修复 DomesticAudience Schema 混乱

**问题**：DeepSeek V4 Flash 在多智能体会话中容易混淆输出格式——当它在同一 session 中先后输出 `AgentResponse`、`Proposal`、`DomesticScore` 三种不同 Schema 时，缓存前缀匹配导致模型将前面 Agent 的 Schema "惯性输出"。

**方案 A（推荐）**：端到端 Schema 消歧——在 DomesticAudience 的 user message 中注入更强的格式约束：

```
修改文件: backend/agents/domestic_audience.py

在 user_message 末尾添加：
"""
⚠️ 【重要格式警告】
你需要输出的是 DomesticScore，不是 AgentResponse 也不是 Proposal！
输出必须且仅包含以下字段：
{
  "agent_type": "strong" (或 "weak"),
  "political_acceptability": <0到1之间的浮点数>,
  "pressure_level": <0到1之间的浮点数>,
  "key_concerns": ["关注点1", "关注点2", ...]
}
禁止输出 action、counter_proposal、territory_split、resource_allocation 等不属于 DomesticScore 的字段！
"""
```

**方案 B（备用）**：如果方案 A 效果不显著，在 `domestic_audience.py` 的 `act` 方法中增加**后处理校验+重试**逻辑——检测 `parsed_output` 中是否存在 `action` 或 `territory_split` 等字段，若有则判定为 Schema 错误并自动重试（修改 user message 加强约束后重发）。

### 5.2 🔴 优先级 P0：降低双方拒绝阈值

**问题**：双方提示词中的"压力 > 0.7 → 不可让步"的硬阈值导致了不可调和的僵局。

**方案**：

```
修改文件: backend/prompts/strong_party.txt

原文: "- 国内政治压力超过0.7时，几乎无法做出任何让步——此时任何让步都可能引发政权危机"
改为: "- 国内政治压力超过0.7时，让步需要强有力的安全保障或战略利益补偿——但最后一轮谈判时，无协议的长期代价可能超过短期政治风险"

原文: "- 你的保留效用约为初始效用的60%——低于此阈值你会选择无协议而非接受"
改为: "- 你的保留效用约为初始效用的30%（最后两轮降至20%）——低于此阈值你才会选择无协议"

修改文件: backend/prompts/weak_party.txt

原文: "- 国内政治压力超过0.7时，领导人几乎无法接受任何不收复领土的协议"
改为: "- 国内政治压力超过0.7时，领导人需要在协议中获得实质性的领土收复承诺或国际监督保障——分阶段收复方案可作为替代路径"

新增（弱方）:
"- 最后一轮谈判：如果拒绝该提案意味着维持更糟糕的现状至少3-5年，请认真考虑接受一个\"虽不完美但可推进\"的协议"
```

```
修改文件: backend/prompts/mediator_pro_strong.txt / mediator_neutral.txt / mediator_pro_weak.txt

在提示词中新增:
"""
## 最后轮次策略
当进入最后 2 轮谈判时（第7-8轮），你必须：
1. 显著调整提案以促成协议——即使偏离你的偏见方向
2. 将边支付金额提升至预算上限的 80-100%
3. 在 justification 中明确告知双方：这是最后一轮，无协议的代价将是长期僵局
"""
```

### 5.3 🟡 优先级 P1：增加边支付的有效性

**问题**：边支付金额太小或为 0，弱方不接受"以经济补偿替代领土主权"。

**方案**：

```
修改文件: backend/config.py

- 将 side_payment_budget_pct 从 0.5 提高到 2.0（从 GDP 的 0.5% 提高到 2%）
  理由：0.5% 的 GDP 对于补偿领土让步而言金额太低，2% 的 GDP 约等于马歇尔计划级别的援助规模，
  足以让弱方国内将"经济重建"作为接受协议的替代叙事

修改文件: backend/prompts/weak_party.txt

修改:
- "边支付如果支付给你，可以作为短期补偿但不能替代领土主权"
改为:
- "边支付如果金额足够大（如 > 1.0 单位），可以作为分阶段收复领土方案的过渡期补偿——这不等于放弃主权，而是为最终收复争取时间和资源"
```

### 5.4 🟡 优先级 P1：优化提示词中的效用数值系统

**问题**：当前效用变化量（utility_change）在智能体推理中被频繁引用，但其数值计算缺乏引导一致性——强方起始效用 300（AR=3.0），最低降至理论上的 180（60% 保留阈值），但这个"效用空间"太宽，导致智能体在中间地带无限反提案。

**方案**：

```
修改文件: backend/prompts/strong_party.txt 和 weak_party.txt

在用户消息（每轮 respond_to_proposal 时）中增加：
"""
## 效用变化指导
- 你当前的效用变化量应该在以下范围：
  - 接受(accept)：utility_change = -5 到 -15（接受意味着一定的让步损失）
  - 拒绝(reject)：utility_change = 0（维持现状）
  - 反提案(counter_proposal)：utility_change = 0 到 -5（你希望对方让步）
- 对照：你的保留效用阈值为初始效用的30%
"""
```
